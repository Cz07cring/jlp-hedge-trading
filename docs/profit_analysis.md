# JLP 中性套利策略收益分析

## 策略原理

**持有 JLP** + **做空 SOL/ETH/BTC** = Delta 中性（对冲价格波动风险）

---

## 收益来源

### ✅ 1. JLP 池子收益（主要收益）
- **来源**: Jupiter 交易所的交易手续费分成（75%）+ SOL 质押收益（~7%）
- **形式**: JLP 价格上涨（virtual_price 增加）
- **预估**: APY 约 15-25%（当前约 20%，根据市场交易量波动）
- **特点**: 牛市时可能降低（交易者盈利），震荡/熊市时较高
- **核算**:
  ```
  JLP收益 = JLP数量 × (当前价格 - 买入价格)
  ```

### ✅ 2. 资金费率收入（做空收益）
- **来源**: 永续合约资金费率（多头付给空头）
- **条件**: 当市场看多时，资金费率为正，做空方收钱
- **频率**: 每 8 小时结算一次
- **核算**:
  ```
  单次费率收入 = 持仓价值 × 资金费率
  日收入 ≈ 持仓价值 × 资金费率 × 3
  ```

### ⚠️ 3. 价格波动损益（应该接近 0）
- **理论**: 对冲后 Delta 中性，价格涨跌不影响
- **实际**: 存在微小偏差（调仓延迟、对冲比例不精确）
- **核算**:
  ```
  波动损益 = JLP价值变化 + 空单盈亏
  理想情况下 ≈ 0
  ```

---

## 成本/支出

### ❌ 1. 交易手续费
- **发生**: 开仓、平仓、调仓时
- **费率**: Maker 约 0.02%，Taker 约 0.05%
- **核算**: 交易金额 × 费率

### ❌ 2. 负资金费率（做空支出）
- **发生**: 当市场看空时，资金费率为负，做空方付钱
- **频率**: 每 8 小时结算一次
- **核算**: 持仓价值 × |负资金费率|

### ❌ 3. 滑点损耗
- **发生**: 大单交易时
- **控制**: 使用 Maker 单 + 拆单策略减少

## 收益核算公式

### 当前系统使用
```
账户净值 = JLP价值 + USDT余额 + 未实现盈亏
累计收益 = 账户净值 - 本金(JLP价值)
         = USDT余额 + 未实现盈亏
```

**解释**:
- 初始状态：全部资金买入 JLP，USDT 余额 = 0
- 运行后：JLP 产生收益 + 资金费率收入 → 转为 USDT 余额
- 未实现盈亏：空单的浮动盈亏（反映真实账户价值）
- 累计收益 = 已实现收益(USDT余额) + 未实现收益(浮动盈亏)

### 详细版（理想情况）
```
累计收益 = JLP池子收益 + 资金费率净收入 - 交易手续费 - 滑点损耗
```

其中：
- JLP池子收益 = JLP数量 × (当前价格 - 初始价格)
- 资金费率净收入 = Σ(持仓价值 × 资金费率) 每8小时
- 交易手续费 = Σ(交易金额 × 费率)

| 资金费率累计 | 需记录每次结算 | 定时采集并累加 |

## 建议

1. **当前够用**: 累计收益 = USDT余额，这个核算方式简单直接
2. **后续优化**: 如果想细分收益来源，需要：
   - 记录初始 JLP 买入价格
   - 每 8 小时记录资金费率结算
   - 记录每笔交易的手续费
3. **关注重点**:
   - 资金费率是否长期为正
   - 对冲比例是否稳定在 100% 左右
   - JLP APR 走势
